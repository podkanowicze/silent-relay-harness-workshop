## New design rule

Every exercise must begin with two applications a normal person immediately recognises:

1. the intended product;
2. a neighbouring product that can be built from the same visible UI primitives.

The drift must be explainable as a concrete chain of code and interface decisions. A clever domain distinction is insufficient if the final screen does not visibly become another familiar product.
